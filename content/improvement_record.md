### Topic 1: Network Security
- **Status**: Implemented
- **Date**: February 3, 2026
- **Details**: Created 50+ practice questions with detailed explanations covering firewall configuration, VPN implementation, network protocols, and security best practices

#### Sample Questions:

**Question 1:** What is the primary purpose of a Demilitarized Zone (DMZ) in network security?
A) To provide high-speed internet access
B) To isolate internal networks from external threats
C) To store backup data
D) To manage user authentication

**Answer:** B) To isolate internal networks from external threats
**Explanation:** A DMZ is a network segment that sits between an internal network and an external network (like the internet), providing an additional layer of security by isolating internal resources from external threats.

**Question 2:** Which VPN protocol is considered the most secure for remote access?
A) PPTP
B) L2TP/IPSec
C) SSTP
D) IKEv2

**Answer:** B) L2TP/IPSec
**Explanation:** L2TP/IPSec combines the Layer 2 Tunneling Protocol with IP Security, providing strong encryption and authentication, making it one of the most secure VPN protocols available.

**Question 3:** What is the main difference between a stateful and stateless firewall?
A) Stateful firewalls are faster than stateless firewalls
B) Stateless firewalls maintain connection state information
C) Stateful firewalls track connection state and make decisions based on context
D) Stateless firewalls are more secure than stateful firewalls

**Answer:** C) Stateful firewalls track connection state and make decisions based on context
**Explanation:** Stateful firewalls maintain information about active connections and make decisions based on the connection state, while stateless firewalls examine individual packets without considering connection context.

**Question 4:** Which of the following is NOT a common type of network attack?
A) DDoS
B) Man-in-the-middle
C) Social engineering
D) Load balancing

**Answer:** D) Load balancing
**Explanation:** Load balancing is a legitimate network technique used to distribute traffic across multiple servers, while DDoS, man-in-the-middle, and social engineering are all types of network attacks.

**Question 5:** What does the acronym ACL stand for in network security?
A) Advanced Control List
B) Access Control List
C) Authentication Control Layer
D) Application Control Library

**Answer:** B) Access Control List
**Explanation:** An Access Control List (ACL) is a list of permissions attached to an object that specifies which users or system processes have access to that object and what operations they can perform.

### Topic 3: Database Management
- **Status**: Implemented
- **Date**: February 3, 2026
- **Details**: Created 50+ practice questions with detailed explanations covering database design, security, administration, and optimization

#### Sample Questions:

**Question 1:** What does the acronym ACID stand for in database transactions?
A) Atomicity, Consistency, Isolation, Durability
B) Authentication, Consistency, Integrity, Durability
C) Atomicity, Confidentiality, Isolation, Data
D) Accuracy, Consistency, Integrity, Dependability

**Answer:** A) Atomicity, Consistency, Isolation, Durability
**Explanation:** ACID is a set of properties that guarantee database transactions are processed reliably: Atomicity (all-or-nothing), Consistency (valid state), Isolation (concurrent transactions), Durability (permanent after commit).

**Question 2:** What is the primary purpose of database normalization?
A) To increase storage capacity
B) To reduce data redundancy and improve data integrity
C) To improve query performance
D) To enhance security

**Answer:** B) To reduce data redundancy and improve data integrity
**Explanation:** Database normalization is the process of organizing data to reduce redundancy and improve data integrity by decomposing tables.

**Question 3:** Which SQL command is used to retrieve data from a database?
A) INSERT
B) UPDATE
C) SELECT
D) DELETE

**Answer:** C) SELECT
**Explanation:** The SELECT statement is used to retrieve data from one or more database tables or views.

**Question 4:** What is a foreign key in a relational database?
A) A key that encrypts data
B) A primary key of another table referenced in the current table
C) A key that is not unique
D) A key used for authentication

**Answer:** B) A primary key of another table referenced in the current table
**Explanation:** A foreign key is a field (or collection of fields) in one table that uniquely identifies a row in another table, establishing a relationship between the tables.

**Question 5:** What is the difference between a clustered and non-clustered index?
A) Clustered indexes are faster than non-clustered indexes
B) A table can have only one clustered index but multiple non-clustered indexes
C) Clustered indexes are used for text data only
D) Non-clustered indexes are more secure

**Answer:** B) A table can have only one clustered index but multiple non-clustered indexes
**Explanation:** A clustered index determines the physical order of data in a table and there can only be one per table, while non-clustered indexes are separate structures that can be created in multiple instances per table.

### Guide 1: Network Connectivity Issues
- **Status**: Implemented
- **Date**: February 3, 2026
- **Details**: Created step-by-step troubleshooting guide for common network connectivity problems with diagnostic tools and solutions

#### Network Connectivity Troubleshooting Guide

**Step 1: Identify the Problem**
- Define the specific connectivity issue
- Determine if it's affecting one device or multiple devices
- Note when the problem started
- Identify any recent changes to the network

**Step 2: Verify Physical Connections**
- Check all cable connections
- Look for LED indicators on network devices
- Verify power connections
- Test with different cables if possible

**Step 3: Check IP Configuration**
- Run `ipconfig` (Windows) or `ifconfig` (Linux/Mac)
- Verify IP address, subnet mask, and default gateway
- Check DNS server settings
- Renew IP address if using DHCP (`ipconfig /release` and `ipconfig /renew`)

**Step 4: Test Basic Connectivity**
- Ping the local loopback address: `ping 127.0.0.1`
- Ping the local IP address: `ping [local-ip]`
- Ping the default gateway: `ping [gateway-ip]`
- Ping an external IP address: `ping 8.8.8.8`

**Step 5: Test DNS Resolution**
- Try to ping a domain name: `ping google.com`
- Use `nslookup` or `dig` to test DNS resolution
- Try different DNS servers if needed

**Step 6: Check Firewall and Security Software**
- Temporarily disable firewall to test
- Check antivirus settings
- Verify security software isn't blocking connections

**Step 7: Advanced Diagnostics**
- Use `tracert` (Windows) or `traceroute` (Linux/Mac) to identify where packets are failing
- Use `netstat` to check active connections
- Check routing table with `route print` (Windows) or `route -n` (Linux/Mac)

**Step 8: Document and Resolve**
- Document the solution for future reference
- Implement permanent fix
- Test connectivity after resolution
